# 논문 확장 방향 제안: Behavior Tree + RL 하이브리드 협동 NPC

*작성자: 박준형 | 2026-04-21*

---

## 0. 한 문단 요약 (TL;DR)

본 연구실은 협동 NPC 학습을 위해 **"① 선행 연구 v1 (5v5 대칭 전투, 파라미터 공유 PPO) → ② 현재 초안 v2 (비대칭 보스 레이드, 역할별 독립 네트워크 + 커리큘럼 학습) → ③ 본 제안 v3 (BT 기반 기믹 인식 + RL 전술 실행 하이브리드)"** 의 세 단계 연구 라인을 구축하였다. 각 단계는 앞 단계에서 드러난 한계를 구조적으로 해결하며 확장된다. 구체적으로, v2 방식을 B안 관측 확장(123차원)과 함께 300,000 에피소드 학습한 결과(v26 실험), **승률 0%, 스태거 기믹 성공 0회** 의 구조적 한계를 확인하였다. 이는 기믹 파훼가 희소 이벤트 특성상 강화학습 탐색으로는 본질적으로 수렴 어렵기 때문이며, 이를 해결하기 위해 **"이산·규칙 명확 = BT, 연속·맥락 의존 = RL"** 원칙의 2계층 하이브리드를 제안한다. 기존 자산(환경, 관찰 공간, 역할별 네트워크) 은 그대로 재활용되며, Phase 1 (Layer 1 BT 구현 및 1~2주 내 실험가능성 검증) → Phase 2 (본 ablation) → Phase 3 (사용자 평가) 로 진행한다.

**Novelty 포지셔닝 (정직)**: BT+RL 하이브리드 자체는 선례가 있음 (AMD Schola 2025, MARL-BT 2023). MMORPG 레이드 RL 환경도 있음 (RaidEnv 2023). 본 연구의 좁은 차별점은 **"텔레그래프 기반 협동 파훼 기믹(SEAL/MARK/STAGGER 등) 특화 + 파티 측 학습 + Pure RL 수렴 실패의 300k ep 정량 실증"** 이다 (상세 비교: Section 4.2, 4.3).

## 1. 연구 진행 경과 — 점진적 발견의 3단계

### 발견 1 ➜ 역할 분리: "파라미터 공유만으로는 비대칭 역할이 학습되지 않는다"

**시도 (v1 = 선행 연구, 5v5 대칭 격자, 파라미터 공유 PPO + 셀프플레이 2단계)**
- 모든 에이전트가 동일한 정책 네트워크를 공유하며 셀프플레이 → 유저 적응형 NPC 2단계로 학습
- 결과: 대칭 환경에서는 유효, 승률 확보됨

**확장 시도 (v10 = 비대칭 보스 레이드 초기 실험, 여전히 파라미터 공유)**
- 보스 레이드(비대칭) 환경으로 이식, 정책 공유 방식 유지
- 결과: 3,000 에피소드에서 승률 13%, **역할 미분화 현상 관찰** — 힐러가 공격만 하고 탱커가 도망치는 등 "무임승차/역할 혼동" 문제
- 원인 분석: 탱커(근접 유지)와 힐러(원거리 유지)는 **최적 행동이 정반대** — 한 네트워크가 둘 다 학습하려다 양쪽 다 어중간하게 수렴

**해결 (v12~v13 = 역할별 독립 네트워크 도입, 본 연구실 paper_draft.md 의 제1 기여)**
- **Actor-Critic 네트워크를 역할별로 완전 분리** — TankNet / HealerNet / SupportNet 각각 독립 PPO
- 각 네트워크는 해당 역할의 경험만 학습 → gradient 간섭 구조적 제거
- 결과: 75,000 에피소드에서 **승률 57.8%** (v13 체크포인트, 보스 HP 900), 뚜렷한 역할 분화 — 힐러 힐량 6배 증가, 탱커가 보스 최근접 유지

### 발견 2 ➜ 점진적 커리큘럼: "고난이도를 직접 학습시키면 아예 수렴하지 않는다"

**시도 (v1~v4 = 초기 실험, 보스 HP 2,500~5,000 직접 학습)**
- 최종 목표 난이도에서 곧바로 학습 시작
- 결과: 10,000 에피소드에도 **승률 0%** — 탐색 과정에서 단 한 번의 승리 경험도 얻지 못해 reward signal 이 완전 부재 → 학습 불가능

**원인 분석**
- 협동 레이드에서 랜덤 탐색만으로 보스를 처치할 확률은 사실상 0 (300턴 × 4에이전트 × 17액션의 조합 공간)
- reward shaping 만으로는 희소한 "승리" 상태로 에이전트를 유도 불가

**해결 (v9 이후 단계적 커리큘럼 도입, paper_draft.md 의 제2 기여)**
- **커리큘럼 학습**: 보스 HP 를 **900 → 1,200 → 2,500** 순차 상승, 각 단계마다 이전 체크포인트를 초기 가중치로 사용 (transfer learning)
- 결과: HP 1,200 단계 진입 직후 이미 승률 10%+ 달성 — 이전 단계에서 학습한 기술(접근, 딜링, 회피) 이 다음 단계로 자연 전이됨
- 이유: 쉬운 보스에서 "이기는 경험" 을 먼저 확보해야 reward 분포가 유의미해지고, 그 위에서 더 어려운 문제를 fine-tune 할 수 있음 (Bengio et al. 2009 커리큘럼 학습 이론과 정합)

### 발견 3 ➜ BT 하이브리드: "희소 기믹 이벤트는 RL 탐색만으로 학습되지 않는다"

**확장 시도 (v26 = 본 연구팀의 최신 실험, 2026-04 진행)**
- v13 기반 + 관찰 공간 확장(123차원 "B안", 패턴 고정 채널·SEAL NPC spot 배정 알고리즘 추가) + 역할별 독립 네트워크 + 커리큘럼 유지
- **300,000 에피소드** (약 17.6시간) 학습
- 결과:

| 지표 | v26 결과 |
|------|---------|
| 최종 승률 | **0%** (최근 5,000 에피소드 연속 wipe) |
| Phase 2 도달률 | 99.0% (페이즈 2 진입은 안정적) |
| **Phase 3 도달률** | **17.4% 피크 (160k ep) → 10.1% (300k ep) 역진** |
| **스태거 기믹 성공** | **0회 / 5,000 에피소드** |
| 평균 보스 딜량 | 51.6% (P2~P3 경계에서 전멸 반복) |

**원인 분석**
1. **기믹 이벤트의 본질적 희소성**: SEAL_BREAK · CURSED_CHAIN · STAGGER 등은 페이즈 전이 시점에만 1~2회 발생. "성공 경험" 이 누적되기 전에 수백 에피소드 실패 → reward signal 부재
2. **Multi-agent credit assignment 어려움**: "누가 덕에 스태거가 성공했는가" 를 개별 에이전트가 귀속 불가 (비정상성)
3. **엔트로피 붕괴**: 160,000 에피소드 이후 정책이 deterministic 해지며 local minimum 에 고착 → 오히려 성능 역진
4. **수동 설계 결함의 발견**: 스태거 게이지 300 / 4턴 / 기여치 10·20·30 설정에서 이론 최대 기여 230 → **학습과 무관하게 수학적으로 불가능**한 설정이었음을 사후 발견. 이는 역설적으로 **"RL 이 수동 설계 결함을 자동 보완하지 못한다"** 는 본 연구의 주장을 강화

**해결 방향 (본 제안서 v3)**
- 기믹 인식은 **이벤트가 이산적이고 규칙이 명확한 문제** → Behavior Tree 가 확정적·검증 가능으로 적합
- 전투 포지셔닝/어그로 관리/힐 타깃 선택은 **연속 공간에서 맥락에 의존하는 문제** → RL 이 적합
- 따라서 **각 문제 유형에 적합한 패러다임을 할당하는 2계층 하이브리드 아키텍처** 를 제안 (Section 3 참조)

### 1.4 본 연구의 위치

위 경과는 **"단일 정책으로 모든 것 학습 → 역할 분리 → 커리큘럼 → 이벤트 유형별 패러다임 분리"** 의 점진적 분해 과정이다. 각 단계가 앞 단계의 구체적 한계를 해결하며, v3 는 이 흐름의 자연스러운 종착점이다. v3 가 성공한다면 3부작(v1 → v2 → v3) 이 **"협동 게임 NPC 개발의 단계별 설계 지침"** 으로 완성된다.

## 2. 문제 분석

### 2.1 Pure RL 이 이 환경에서 어려운 근본 이유

게임 AI 문헌(AlphaStar, OpenAI Five)의 공통 관찰: **"규칙이 명확한 구조적 의사결정은 학습보다 스크립팅이 효율적"**.

본 환경의 의사결정 문제를 두 부류로 분해하면:

| 문제 유형 | 성격 | RL 적합성 | 예시 |
|-----------|------|----------|------|
| **A. 기믹 인식 대응** | 이벤트 이산, 규칙 명확, 실패 시 치명적 | 낮음 | "MARK 대상이 나 → 도망" "SEAL 발동 → 내 자리 점유" |
| **B. 전술 실행** | 연속 공간, 문맥 의존, 점진적 최적화 | 높음 | 어그로 관리, 힐 타깃 선택, 공격 타이밍 |

지금까지 진행한 v2 방향(paper_draft 초안, 개인 연구) 은 A와 B를 **동일한 정책 네트워크**에 맡겼다. RL은 B에 자연스럽게 적합하나, A는 학습 신호 희소성 때문에 수렴 못함.

### 2.2 핵심 통찰

> **"각 문제 유형에 적합한 패러다임을 할당하고, 이 둘을 원칙적으로 결합한다."**

이는 AlphaStar 의 "macro action 하드코딩 + 미세 제어 학습" 구조, Hierarchical RL 의 "option framework" 와 같은 계열의 접근이다.

## 3. 제안: 2계층 하이브리드 아키텍처

### 3.1 전체 구조

```
                         [관찰 obs, 123 dim]
                                  ↓
               ┌──────────────────────────────────┐
               │   Layer 1: 기믹 인식 BT          │
               │   (결정론적 규칙)                │
               │                                   │
               │   SEAL 활성 ? → 배정 spot 이동    │
               │   MARK 대상 ? → 반대 방향        │
               │   CHAIN 대상 ? → 파트너 접근     │
               │   CROSS 활성 ? → 안전 사분면     │
               │   STAGGER 창 ? → 보스 근접 딜    │
               │   (기믹 없음) → fall-through     │
               └──────────────────────────────────┘
                                  ↓ (기믹 없을 때만)
               ┌──────────────────────────────────┐
               │   Layer 2: 역할별 RL 전술 실행    │
               │   (PPO)                           │
               │                                   │
               │   TankNet / HealerNet / SupportNet│
               │   학습: 포지셔닝, 공격 타이밍,    │
               │         어그로 관리, 힐 타깃      │
               └──────────────────────────────────┘
                                  ↓
                        [액션 17개 중 선택]
```

### 3.2 Layer 1: BT 기믹 인식 계층

8개 보스 패턴 각각에 대해 **"대상 / 조건 / 행동 / 종료"** 4요소로 정의된 규칙 테이블. 이미 `FSMNpcPolicy`, `SEAL_BREAK spot assignment` 알고리즘이 구현되어 있어 **구현 비용 최소**.

| 패턴 | BT 규칙 |
|------|---------|
| SLASH / TAIL_SWIPE | 현재 위험 영역 내이면 escape_dx/dy 방향으로 이동 |
| CHARGE | 직선 경로 수직 방향으로 2칸 이동 |
| ERUPTION | 가장 가까운 원 밖으로 이동 |
| MARK | 대상이 나 → 다른 아군 반대 방향 5m+ / 대상이 남 → carrier 반대편 |
| STAGGER | 보스 근접 사거리 확보 + ATTACK_SKILL 우선 |
| CROSS_INFERNO | 가장 가까운 안전 사분면 중심으로 이동 |
| CURSED_CHAIN | 파트너 위치로 접근 (3m 이내 유지) |
| SEAL_BREAK | `tg.extra["npc_spots"][my_uid]` 에서 배정 spot 조회 → 이동 |

### 3.3 Layer 2: 역할별 RL 전술 계층

- 현재 v2 의 **B안 관측 구조 (123 dim) 그대로 활용**
- 역할별 독립 PPO 네트워크 (TankNet / HealerNet / SupportNet)
- 보상 설계 단순화: **기믹 성공 보상을 제거**하고 순수 전투 보상(딜·어그로·힐·생존)만 남김 → 학습 공간 축소 → 수렴 가속

### 3.4 기존 자산 재활용

| 자산 | 출처 | v3 에서의 역할 |
|------|------|---------------|
| BossRaidEnv (환경) | v2 | 그대로 |
| B안 관측 구조 (123 dim) | v2 | Layer 2 입력 |
| 역할별 독립 PPO | v2 | Layer 2 엔진 |
| SEAL spot assignment 알고리즘 | v2 | Layer 1 BT 에서 활용 |
| FSMNpcPolicy | v2 | Layer 1 BT 의 골격 |
| 3단 계층 보상 | v2 | Layer 2 의 전투 보상만 유지 |

→ **새로 구현해야 할 것은 Layer 1 BT 트리(1~2일 분량) 와 두 계층을 연결하는 dispatcher 뿐**.

### 3.5 동작 시나리오 — 상황별 BT/RL 의사결정 흐름

매 턴 각 NPC 는 `HybridPolicy.act()` 호출. 내부 흐름:

```
   BT 규칙 6개를 우선순위대로 체크
        ↓
   ┌── 규칙 fire → int 액션 반환 ─────── RL 호출 안 함
   │
   └── 아무 규칙도 fire 안 함 → None ── RL.act() 호출
```

**시나리오별 기대 동작**:

| # | 상황 | BT 결과 | 누가 결정? | 기대 행동 |
|---|------|---------|-----------|----------|
| A | 보스 주변, 텔레그래프 없음 | 모든 규칙 pass → None | **RL** | 역할별 전술 (탱커 근접 유지, 힐러 포지셔닝, 공격 타이밍) |
| B | SLASH 5턴 남음, 내가 영역 안 | 임박(≤2턴) 아님 → None | **RL** | RL 이 danger sensor 보고 선제 이동 (여유 있음) |
| C | SLASH 2턴 남음, 내가 영역 안 | 임박 위험 규칙 fire | **BT** | 8방향 중 가장 빠른 탈출 방향 |
| D | MARK wind_up, 내가 표식자 | MARK 규칙 fire | **BT** | 아군 중심 반대 방향 (escape distance 5m 이상) |
| E | MARK, 이미 충분히 멀어짐 | already safe → None | **RL** | RL 이 딜·포지셔닝 계속 |
| F | STAGGER 창 열림, 보스 사거리 안 | STAGGER 규칙 fire | **BT** | 탱커는 TAUNT, 나머지는 스킬/기본 |
| G | SEAL_BREAK 활성 | SEAL 규칙 fire (최우선) | **BT** | 내 배정 spot 이동 → 도착 시 STAY/공격 |
| H | CROSS_INFERNO 4턴 남음, 위험 사분면 | CROSS 규칙 fire | **BT** | 가장 가까운 안전 사분면 중심 이동 |
| I | CHAIN 활성, 파트너 4m+ 떨어짐 | CHAIN 규칙 fire | **BT** | 파트너 쪽 접근 |
| J | CHAIN, 파트너 2m 이내 | 가까움 → None | **RL** | 근접 유지하며 RL 이 딜 수행 |

### 3.6 예상 실행 비율

방금 72턴 에피소드 sanity 테스트 결과, NPC 별 **BT 발동 비율 73~83%**, **RL 17~27%**. 의미:

- **BT 가 "생사 가르는 결정"** (기믹 파훼, 임박 위험 회피) 대부분을 가져감 → 협동 기믹 학습 문제 해소
- **RL 은 "뉘앙스 있는 전술"** (어그로 관리, 힐 타깃 선택, 공격 선택) 을 담당 → v2 의 역할별 정책이 그대로 유효

두 계층이 **상호배타적이고 상호보완적** 이어서 credit assignment 단순화 효과. 이 분업 구조가 Phase 2 ablation 의 핵심 측정 대상이다.

## 4. 학술적 의의

### 4.1 연구 라인상 위치

**3부작 완성** 관점에서의 프레이밍:

| 단계 | 제목 (가상) | 핵심 질문 | 핵심 기여 |
|------|------------|----------|----------|
| **v1** (선행 연구, 5v5 대칭 전투) | "Self-Play 파라미터 공유로 학습하는 대칭 협동 NPC" | 대칭 전투에서 협동 NPC 가능한가? | 파라미터 공유 + 셀프플레이 2단계 |
| **v2** (현재 초안, 1보스 4파티 비대칭) | "역할 이질성 환경에서의 역할별 독립 강화학습" | 역할 이질성이 크면 어떻게 해야 하는가? | 역할별 독립 네트워크 + 3단 계층 보상 + 커리큘럼 학습 |
| **v3** (본 제안서) | "희소 기믹 환경을 위한 BT + RL 하이브리드 아키텍처" | 기믹 파훼가 핵심인 환경에서는? | **BT 기반 인식 계층 + RL 전술 실행 계층** |

각 논문이 **앞 논문의 적용 범위를 규명하면서 확장** — 부정이 아닌 포괄 관계. v2가 v1을 대체하지 않고 "비대칭 환경에서는 역할 분리가 필요"로 범위 한정했듯, v3도 "희소 기믹 환경에서는 BT 결합이 필요"로 확장.

### 4.2 Novelty — 정직한 경쟁구별화


#### 4.2.1 협동 파훼 텔레그래프 기믹 특화

**선행 연구의 한계**: 기존 MMORPG RL 벤치마크인 **RaidEnv (arXiv 2307.01676, 2023)** 도 Unity 기반 3인 탱커/딜러/힐러 협동 레이드 환경을 제공하나, **보스 정책이 매우 단순** ("가장 가까운 플레이어 이동 후 공격"). 플레이어 측이 대응·파훼해야 하는 **텔레그래프 기반 기믹** (부채꼴 베기·돌진·장판·봉인해제·집결 스태거 등) 은 포함되지 않음.

**본 연구**: **8개 복합 텔레그래프 기믹 패턴** 을 포함하며, 그 중 4개(MARK, STAGGER, CROSS_INFERNO, CURSED_CHAIN, SEAL_BREAK) 는 **2명 이상의 협동 파훼가 필수**. 이 영역은 RaidEnv 계열 환경으로는 평가 자체가 불가능.

#### 4.2.2 파티 측 학습 (보스 측 아님)

**선행 연구**: WoW 보스 전투 RL (UPC 석사논문) 은 **보스 측 AI** 가 기믹 선택을 학습. 플레이어는 사람.

**본 연구**: **파티 측 NPC 3명이 협동 기믹 파훼를 학습**. 플레이어는 딜러 슬롯. 학습 방향이 정반대이며, 인간 플레이어와 함께 플레이할 수 있는 **협동 NPC 제작** 이라는 산업 응용 목표.

#### 4.2.3 Pure RL 실패의 정량 실증

**선행 연구**: Pure RL 이 협동 기믹 환경에서 수렴하지 못하는 사례를 **장시간 학습 로그로 공개** 한 연구는 검색 한도 내에서 희소.

**본 연구**: **v26 (B안 관측 + 300,000 에피소드 = 17.6시간) 학습 로그** — 승률 0%, 스태거 성공 0/5,000ep, Phase 3 도달률 17.4% → 10.1% 역진 등을 **empirical evidence** 로 제시. 후속 연구자가 "협동 기믹 파훼 RL 의 한계선" 을 정량 참조 가능.

#### 4.2.4 3 기법의 누적적 통합

**선행 연구**: 역할별 독립 네트워크, 커리큘럼 학습, BT+RL 하이브리드는 **각각 독립적으로** 존재.

**본 연구**: 세 기법이 **하나의 협동 레이드 NPC 시스템에 누적적으로 통합**. v1 (파라미터 공유) → v2 (역할 분리 + 커리큘럼) → v3 (BT+RL) 의 **3부작 누적 연구 라인** 으로, 각 단계의 실패 데이터가 다음 단계의 evidence. 이 통합 구조 자체가 설계 공헌.

#### 4.2.5 분해 원칙의 명시화 (저널 단계 추가 검증 필요)

**희망 주장**: **"이벤트 희소성 × 규칙 명확성"** 판정 기반 BT/RL 분해 프레임워크.

**주의**: 이 주장이 실제로 기존 hierarchical RL / hybrid 논문에서 언급되지 않은 novel 제안인지, 저널 투고 단계에서 **더 정밀한 문헌 조사** 필요. AMD Schola (2025), MARL-BT (2023), 그리고 HAVEN (AAAI 2022) 등이 유사한 계층 분해 원칙을 암묵적으로 사용할 수 있음.

### 4.3 인용 가능 선례 및 차별점 (업데이트)

| 선행 연구 | 우리와 유사 | 결정적 차이 |
|----------|-----------|------------|
| **AMD Schola + BT+RL** (arXiv 2510.14154, 2025) | BT+RL 하이브리드 NPC | **단일 에이전트**, multi-task 스킬 중심. 협동 레이드·텔레그래프 기믹 없음 |
| **MARL-BT** (Springer 2023, Complex & Intelligent Systems) | MARL 을 BT 에 임베딩, 협동 11v11 | **Google Research Football** 도메인. 페이즈 전이·희소 기믹 이벤트 없음 |
| **RaidEnv** (arXiv 2307.01676, 2023) | Unity 기반 MMORPG 레이드, 탱커/딜러/힐러 역할, 3인 협동 | **BT 미사용**, 보스 정책 단순 ("가장 가까운 플레이어 공격"), **텔레그래프 기믹 없음**. 환경 벤치마크 논문 (학습 알고리즘 논문 아님) |
| **WoW Boss RL** (UPC 석사논문) | 보스 전투 RL + 사전 스크립팅 혼합 | **보스 측 AI** (우리는 파티 측). 단일 에이전트. 방향 반대 |
| **Mini Honor of Kings** (arXiv 2406.03978, 2024) | MARL 보스 전투 환경 (여러 영웅 vs Dark Dragon) | 텔레그래프 파훼 기믹 없음, 전술적 딜 경쟁 중심 |
| **HAVEN** (AAAI 2022) | Hierarchical MARL with dual coordination | BT 미사용, 범용 MARL 프레임워크 |
| **AlphaStar** (Nature 2019) | Macro action 일부 스크립팅 + 학습된 마이크로 정책 | RTS(StarCraft II), 대칭 경쟁 환경. 분해 원칙은 암묵적 |
| **OpenAI Five** (arXiv 2019) | 아이템 구매 등 일부 규칙화 + 학습 | MOBA(Dota 2), 5v5 대칭 경쟁 |

**종합**: 우리의 기여는 "BT+RL을 세계 최초로 결합" 이 아니라, **"텔레그래프 기반 협동 파훼 기믹(SEAL/MARK/STAGGER 등)이 핵심인 MMORPG 레이드 환경에서의 파티 측 NPC 학습"** 이라는 좁고 구체적인 조합에 대한 시스템 설계 + 실증 데이터 기여.

## 5. 검증 계획 (3단계)

### Phase 1 — Layer 1 BT 구현 + 실험 가능성 검증 *(1~2주)*

**목표**: "하이브리드가 pure RL 대비 승률을 유의미하게 올릴 수 있는가" 신속 확인.

**할 일**:
1. Layer 1 BT (8 패턴 규칙) 구현 — `src/boss/bt_policy.py`
2. Dispatcher (BT → RL fall-through) 구현 — `src/boss/hybrid_policy.py`
3. Layer 2는 **v13 체크포인트(57.8% 승률)를 그대로 재사용** — 추가 학습 없이 결합 효과만 측정

**실험** (각 500 에피소드) — **핵심 검증 가설**: "하이브리드 승률이 Pure RL 과 Pure BT 각각보다 유의미하게 높다"

| 조건 | 예상 승률 | 예상 스태거 성공률 | 예상 SEAL 성공률 | 근거 |
|------|---------|------------------|-----------------|------|
| Pure RL (v26 체크포인트 = B안 관측 + 300k ep) | **0%** | 0% | ~10% | v26 실측 (최근 5,000 ep 모두 wipe) |
| Pure RL (v13 체크포인트 = 역할별 독립 PPO, 75k ep, HP 900) | ~57% | 저조 | 저조 | paper_draft 기록값 |
| Pure BT (Layer 1 규칙 + 전투도 스크립트) | 40~55% | **60%+** | **70%+** | 기믹은 잘 처리하나 어그로·포지셔닝 어설픔 |
| **Hybrid (Layer 1 BT + Layer 2 = v13 RL)** | **70%+** | **70%+** | **80%+** | v13 의 전투 능력 + BT 의 기믹 커버리지 합 |
| Hybrid (Layer 1 BT + Layer 2 = v26 RL) | 60%+ | 70%+ | 80%+ | v26 이 승률 0 이어도 포지셔닝 특성 일부 학습됨 |

**예상 BT vs RL 발동 비율** (실측 sanity 기준): BT 73~83%, RL 17~27%. 두 계층이 상호보완.

**마일스톤**: Hybrid 가 Pure RL 대비 승률 **+10%p 이상** AND 스태거 성공률 +30%p 이상 이면 Phase 2 진행 확정.

### Phase 2 — 본 실험 *(2~3주)*

**Ablation 5 조건** × 각 조건 100,000 에피소드 학습 + 500 에피소드 평가:

| # | 조건 | 설명 | 평가 지표 |
|---|------|------|----------|
| 1 | Pure RL (v26 = B안 관측 확장 설정 재현) | 우리의 최신 실패 사례, 하이브리드의 baseline | 승률, 기믹 성공률, 생존 스텝, DPS, 역할 분화도 |
| 2 | Pure FSM (전체 하드코딩) | 규칙 기반의 상한선 | 동일 |
| 3 | **Hybrid (제안, Layer 1 BT + Layer 2 RL)** | **본 연구의 제안 방식** | 동일 |
| 4 | Hybrid w/o coop rules (SEAL/CHAIN만 BT, 나머지 패턴은 RL) | 각 기믹별 BT 기여 분리 측정 | 동일 |
| 5 | Hybrid w/o RL layer (전투까지 전부 FSM) | Layer 2 RL 의 전술 기여 측정 | 동일 |

**정량 분석**:
- 각 기믹 패턴별 파훼 성공률 비교
- 페이즈별 생존률 곡선
- 보스 딜량 분포
- 역할별 행동 히스토그램 (탱커는 어그로 유지 %, 힐러는 힐 %)

### Phase 3 — 사용자 평가 *(2주)*

기존 `paper_draft.md` 의 4.4 절 계획 그대로 실시:
- 피험자: N=16~24 (서강대 가상융합전문대학원생)
- 설계: Within-subject, 순서 카운터밸런싱
- 3 조건: Pure FSM (규칙만) / Pure RL (v26 설정, B안 관측 + 300k ep 학습된 정책) / **Hybrid (제안)**
- 측정: GEQ (몰입), Trust in Automation (Jian et al.), NASA-TLX (인지 부하)
- 주관 인터뷰 핵심 질문: **"어느 NPC 팀이 사람 팀원 같았는가"**
